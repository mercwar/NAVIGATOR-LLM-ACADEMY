Chapter 2: Security and Privacy Considerations

Page 7 – Security and Privacy

As LLMs become integrated into workflows, security and privacy must be prioritized. Users often share sensitive information, and understanding how models handle data is critical for safe usage.

Key Concerns

  • Data Retention – Know whether prompts and outputs are stored.
  • Confidentiality – Avoid sharing private or proprietary information.
  • Access Control – Ensure only authorized users can interact with LLMs.
  • Bias and Ethics – Be aware of potential biases in model outputs.

Organizations should establish policies for LLM use, including guidelines on what information can be shared and how outputs should be verified.

Best Practices

  • Use LLMs in secure environments with encryption.
  • Regularly audit usage to detect risks.
  • Educate teams on safe prompt design.
  • Verify outputs before relying on them for critical decisions.

Practical Exercise

  • Write a prompt that includes sensitive data (e.g., a client’s name).
  • Revise the prompt to remove or anonymize the data.
  • Compare the outputs and note how anonymization affects results.

By practicing secure usage, you’ll protect both personal and organizational data while still benefiting from the power of LLMs.
